## Project Overview

**OptiCrop** is a smart agricultural production optimization engine that leverages Machine Learning to provide intelligent, data-driven recommendations to farmers, agronomists, and policymakers. By integrating key environmental factors — Nitrogen (N), Phosphorous (P), Potassium (K), Temperature, Humidity, pH, and Rainfall — OptiCrop recommends the most suitable crop, assesses crop compatibility, detects plant diseases, and provides fertilizer guidance through a responsive local web application.

The platform is built using **pure Python** (no external web framework), runs locally at `http://127.0.0.1:8000`, and requires no API keys or cloud services to operate.

---

## Three Core Scenarios

| Scenario | Title | Description |
|----------|-------|-------------|
| **Scenario 1** | Smart Crop Recommendation | Farmer enters N, P, K, temperature, humidity, pH, and rainfall → System recommends the best crop from 22 options using an ML ensemble (Random Forest + Decision Tree + SVC) with confidence score |
| **Scenario 2** | Crop Suitability & Environmental Assessment | User selects a specific crop → System evaluates all 7 environmental metrics against ideal ranges and returns a weighted suitability score with remediation steps |
| **Scenario 3** | Agricultural Research & Policy Planning | Researcher uses the analytics dashboard to explore crop-environment patterns, filter by crop/region/season, and export data as CSV for evidence-based policy planning |

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/recommend` | Crop recommendation — 9 features → predicted crop + confidence |
| POST | `/api/suitability` | Crop suitability — 7 metrics → weighted score + remediation |
| POST | `/api/fertilizer` | Fertilizer advice — NPK gap analysis → type + dosage |
| POST | `/api/disease` | Symptom disease detection → disease name + risk level |
| POST | `/api/image-disease` | Image disease detection → upload leaf → disease class |
| GET  | `/api/weather` | Real-time weather → city/GPS → temperature, humidity, rainfall |
| GET  | `/api/analytics` | Analytics summary → crop/region/season stats for dashboard |

## Setup & Run Instructions

### Step 1 — Clone the Repository
```bash
git clone https://github.com/RamanHarshitha/Smart_Bridge
cd Smart_Bridge-master
```

### Step 2 — Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 3 — Download the Dataset
Download `Crop_recommendation.csv` from Kaggle:
https://www.kaggle.com/datasets/atharvaingle/crop-recommendation-dataset

Place it in the `data/` folder as `data/Crop_recommendation.csv`.

### Step 4 — Train the Crop Recommendation Model
```bash
python3 ml/train_model.py
# Output: models/crop_model.joblib + models/confusion_matrix.png
```

### Step 5 — Train the Disease Model
```bash
python3 ml/train_disease_model.py
# Output: models/disease_model.joblib + models/disease_confusion_matrix.png
```

### Step 6 — Run the Server
```bash
python3 server.py
# Server starts at: http://127.0.0.1:8000
# Also accessible on LAN at: http://192.168.x.x:8000
```

### Step 7 — Open in Browser
Navigate to `http://127.0.0.1:8000` in any modern browser (Chrome, Firefox, Edge).
